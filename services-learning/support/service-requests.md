# Service requests

Some NI support paths require a valid service agreement. GitBook can make this transparent without hiding general product documentation.

## Before opening a request

{% stepper %}
{% step %}
Collect product name, serial number, software version, and operating system.
{% endstep %}

{% step %}
Check product documentation, release notes, and KnowledgeBase content.
{% endstep %}

{% step %}
Attach logs, screenshots, code snippets, or test results that reproduce the issue.
{% endstep %}

{% step %}
Open a service request from the NI support portal.
{% endstep %}
{% endstepper %}
