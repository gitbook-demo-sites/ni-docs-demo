# API conventions

For NI developer documentation, GitBook should separate narrative guidance from generated reference pages.

| Topic | Recommended docs pattern |
| --- | --- |
| Endpoint contracts | OpenAPI generated reference |
| Authentication and scopes | Hand-authored guide |
| Errors and pagination | Shared convention page |
| SDK examples | Tabs by language |
| Version policy | Versioned pages or variants |

{% hint style="info" icon="code-branch" %}
Generated API references reduce drift between implementation and documentation.
{% endhint %}
